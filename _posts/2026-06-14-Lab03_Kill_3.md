## Paso 1: Confirmar la vulnerabilidad
Aplicamos La técnica T1046: objetivo principal es mapear la superficie de ataque interna para encontrar sistemas vulnerables y planificar el movimiento lateral.
![imagen](/assets/imagenes/lab03/vulneravilidad.jpg){: width="700" height="400" }

## Paso 2: Explotar con Metasploit
Aplicamos La técnica T1210: diferencia del descubrimiento, aquí el atacante no solo busca, sino que ataca directamente un servicio expuesto para saltar de una máquina comprometida a otra dentro de la red.
![imagen](/assets/imagenes/lab03/paso02.jpg){: width="700" height="400" }

inportante tener presente las IPs de LHOST y RHOSTS , debdido que tenemos que tener presente la victima y el atacante.

en este caso tenemos:

| Máquina | IP asignada |
|:--------|:------------|
| Kali Linux (atacante) | 10.0.2.5 | 
| Metasploitable 3 (víctima) | 10.0.2.4 |

## Paso 3: Post-Explotación con Meterpreter
Aplicamos La técnica T1068:el método por el cual un atacante aprovecha una vulnerabilidad de software en un sistema operativo, kernel, driver o aplicación local para elevar sus permisos actuales.A diferencia de la técnica anterior (T1210), que se ejecuta a través de la red para saltar a otra máquina, la T1068 se ejecuta localmente dentro de una máquina donde el atacante ya tiene un acceso inicial limitado.
![imagen](/assets/imagenes/lab03/kill_3_meterpreter.jpg){: width="700" height="400" }

El comando "info" se utiliza para obtener detalles específicos sobre el módulo de Metasploit que estás ejecutando actualmente o sobre las capacidades de la sesión activa.
![imagen](/assets/imagenes/lab03/meter_info.jpg){: width="700" height="400" }

El comando netstat es una de las herramientas más comunes para realizar la técnica T1046 — Network Service Discovery de forma pasiva en el host objetivo.Al ejecutarlo,te mostrará un listado detallado de todas las conexiones de red activas, puertos abiertos y sockets escuchando en la máquina comprometida.
![imagen](/assets/imagenes/lab03/netstast.jpg){: width="700" height="400" }

Su función de comando "hashdump" es extraer los hashes de las contraseñas de las cuentas de usuario locales almacenadas en el sistema objetivo.
![imagen](/assets/imagenes/lab03/hash.jpg){: width="700" height="400" }

En Windows: Meterpreter intenta leer directamente la colmena SAM (Security Account Manager) desde la memoria del proceso lsass.exe o del registro del sistema.
